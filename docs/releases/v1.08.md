# Token Saver Loop 1.08

Portable workflow kit for AI coding agents: strong reviewer models plan and accept, lower-cost worker models execute bounded rounds and collect evidence.

## Highlights

- Portable no-install workflow kit: copy `portable/token-saver-kit/` into any project.
- Reviewer/worker split for model-agnostic AI coding workflows.
- Stable `LATEST_WORKER_PROMPT.md` worker handoff path.
- Round evidence files: `worker_report.json`, `tests.txt`, `diffstat.txt`, and `worker_log.md`.
- `round_status.json` lifecycle marker to avoid reviewing half-written reports.
- Evidence-shaped acceptance guidance that separates implemented behavior from validated behavior.
- Setup doctor, red-flag checks, review summaries, token usage metrics, and project-memory helpers.
- Beginner guide and minimal inspect-only starter task.

## Install-Free Usage

From a target project root on Windows CMD:

```cmd
rmdir /S /Q "%TEMP%\token-saver-loop-kit" 2>NUL & git clone --depth 1 --branch v1.08 https://github.com/ningbo00/token-saver-loop.git "%TEMP%\token-saver-loop-kit" && xcopy "%TEMP%\token-saver-loop-kit\portable\token-saver-kit" "token-saver-kit" /E /I /Y && rmdir /S /Q "%TEMP%\token-saver-loop-kit"
```

Then tell the reviewer model:

```text
Read token-saver-kit/START_HERE.md and act as reviewer only.
```

## Good Fit

- Repo exploration where premium-model context gets expensive.
- Iterative debugging where the worker can absorb retries.
- Bulk edits and docs/i18n drafts that need compact review.
- Teams or solo developers who want durable local evidence instead of long chat history.

## Reviewer Note

Tool verdicts such as `PASS`, `FIX_SAME_TIER`, `DOWNGRADE`, and `STOP` are evidence verdicts only. The reviewer/user still owns final quality acceptance.
